Instrucciones de Configuración

$ cat network_setup.txt

Para que el laboratorio funcione correctamente, sigue estos pasos:

  1. Importa ambas máquinas (*.deb) en **VirtualBox**.
  2. En la configuración de red de ambas máquinas, selecciona el modo **Red Interna** (Internal Network).
  3. Asegúrate de que ambas máquinas están configuradas con las siguientes IPs estáticas **dentro del sistema operativo**:
    • **Máquina Vulnerable:** IP: **10.33.206.2**
    • **Máquina Atacante:** IP: **10.33.206.10**
  4. Una vez configuradas, la Máquina Atacante debería poder hacer ping a la Máquina Vulnerable.

Total: 5 vulnerabilidades significativas encontradas.

  • [ALTO] Ataques de Diccionario

    Permite a un atacante intentar una gran cantidad de contraseñas predefinidas.

  • [ALTO] Ataques de Denegación de Servicio (DoS)

    Incluido para el aprendizaje de técnicas de defensa y ejecución. (Nota: No es una vulnerabilidad de software per se, sino una técnica de ataque incluida para la práctica).

  • [ALTO] Inyección de SQL y Exposición de Contraseñas

    La aplicación es vulnerable a Inyección de SQL, permitiendo la extracción de datos sensibles, incluyendo contraseñas.

  • [ALTO] Exposición de Información Sensible

    Ciertos datos confidenciales (configuraciones, rutas, etc.) son accesibles sin la debida autenticación o protección.

  • [ALTO] Vulnerabilidad en el Almacenamiento de Contraseñas

    Las contraseñas no se almacenan de forma segura (e.g., hash débil o sin salting), facilitando su descifrado.
